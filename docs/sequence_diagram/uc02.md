### 3.3.2.2. Use case Xem danh sách log

| Thuộc tính | Nội dung |
|------------|-----------|
| **Use case ID** | UC_02 |
| **Tên use case** | Xem danh sách log |
| **Mô tả** | Người dùng thực hiện tìm kiếm, lọc và phân tích danh sách log trên giao diện Dashboard dựa theo các tiêu chí như môi trường, dịch vụ, cấp độ lỗi hoặc cú pháp tìm kiếm nâng cao. |
| **Tác nhân** | Lập trình viên (Developer), Kiểm thử viên (Tester) |
| **Tiền điều kiện** | - Người dùng đã truy cập vào trang **Log Explorer**. |
| **Hậu điều kiện** | - Danh sách log tương ứng với bộ lọc được hiển thị trực quan trên giao diện. |

#### Luồng cơ bản

| Hành động của tác nhân | Phản hồi của hệ thống |
|------------------------|-----------------------|
| **1.** Người dùng thiết lập các bộ lọc tìm kiếm như môi trường, tên ứng dụng (service), cấp độ log (ERROR, WARN, INFO) hoặc nhập từ khóa tìm kiếm nâng cao. | |
| | **2.** Giao diện cập nhật trạng thái bộ lọc và gửi yêu cầu API kèm theo các tham số tìm kiếm đến Backend. |
| | **3.** Backend chuyển đổi các tham số tìm kiếm thành câu truy vấn Elasticsearch DSL và gửi yêu cầu truy vấn tới Elasticsearch. |
| | **4.** Elasticsearch tìm kiếm trên các chỉ mục log phù hợp và trả về tập dữ liệu log thỏa điều kiện kèm thông tin phân trang cursor-based. |
| | **5.** Backend ánh xạ dữ liệu từ Elasticsearch sang DTO, đóng gói kết quả dưới dạng JSON và trả về cho Frontend. |
| | **6.** Giao diện hiển thị danh sách log dưới dạng bảng hoặc dòng thời gian, cho phép người dùng xem chi tiết log và tiếp tục tải thêm dữ liệu khi cần. |

#### Luồng thay thế

| Mã | Mô tả |
|-----|--------|
| **A1** | Người dùng chỉ chọn một phần bộ lọc (ví dụ chỉ chọn environment hoặc chỉ nhập từ khóa). Hệ thống vẫn thực hiện truy vấn với các tiêu chí hiện có. |
| **A2** | Người dùng không nhập từ khóa tìm kiếm. Hệ thống chỉ lọc theo các tiêu chí cấu trúc như environment, app, service và logLevel. |
| **A3** | Người dùng mở một log cụ thể từ danh sách. Hệ thống hiển thị thông tin chi tiết của log trong khung xem chi tiết. |

#### Luồng ngoại lệ

| Mã | Mô tả |
|-----|--------|
| **E1** | Nếu Backend không truy vấn được Elasticsearch, hệ thống trả về lỗi và Frontend hiển thị thông báo tải log thất bại. |
| **E2** | Nếu bộ lọc hoặc câu truy vấn nâng cao không hợp lệ, hệ thống không trả về kết quả và hiển thị thông báo lỗi truy vấn. |
| **E3** | Nếu không có log nào thỏa mãn điều kiện tìm kiếm, hệ thống hiển thị danh sách rỗng thay vì báo lỗi. |

```
@startuml


box "Client" #LightGreen
    participant "Frontend Dashboard" as UI
end box

box "Backend" #LightYellow
    participant "Query Service" as QS
end box

box "Storage" #LightBlue
    database "Elasticsearch" as ES
end box

== Bước 1: Thiết lập bộ lọc ==

UI -> UI: Người dùng chọn bộ lọc\n(environment, app, service,\nlog level, từ khóa)

== Bước 2: Truy vấn log ==

UI -> QS: GET /api/v1/logs\n(kèm tham số tìm kiếm)

activate QS

QS -> QS: Xây dựng truy vấn Elasticsearch DSL

QS -> ES: Thực hiện tìm kiếm log

activate ES

ES --> QS: Danh sách log + thông tin phân trang

deactivate ES

QS -> QS: Chuyển đổi kết quả\nvà đóng gói phản hồi

QS --> UI: Trả về danh sách log

deactivate QS

== Bước 3: Hiển thị kết quả ==

UI -> UI: Hiển thị danh sách log\nvà cho phép xem chi tiết

alt Không có dữ liệu phù hợp

    QS --> UI: Danh sách rỗng

else Lỗi truy vấn

    QS --> UI: Thông báo lỗi

end

@enduml
```
